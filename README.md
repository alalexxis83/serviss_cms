# RepairCMS - Система учета ремонтов

CMS для приема техники в ремонт в стиле Texmobile — оранжевая тема с техническим фоном.

![Style](https://img.shields.io/badge/style-Texmobile-orange)
![PHP](https://img.shields.io/badge/PHP-7.4+-blue)
![MySQL](https://img.shields.io/badge/MySQL-5.7+-blue)

## Скриншоты

- 🎨 Оранжевая цветовая схема как на texmobile.lv
- 📐 Технический чертежный фон
- 📱 Адаптивный дизайн
- ⚡ Современный интерфейс

## Возможности

- 🔐 Авторизация пользователей с ролями
- 📋 Создание заявок на ремонт
- 🔍 Просмотр и редактирование ремонтов
- 📊 История изменений статусов (timeline)
- 🏷️ Управление статусами и приоритетами
- 🖨️ Печать квитанций
- 📊 Статистика на главной

## Требования

- PHP 7.4 или выше
- MySQL 5.7 или выше
- Apache/Nginx с поддержкой mod_rewrite

## Установка

### 1. Настройка базы данных

Откройте файл `config/database.php` и укажите данные для подключения:

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'repair_cms');
define('DB_USER', 'root');
define('DB_PASS', 'ваш_пароль');
```

### 2. Создание базы данных

Создайте базу данных в MySQL:

```sql
CREATE DATABASE repair_cms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. Установка таблиц

Откройте в браузере: `http://ваш-сайт/install.php`

Система автоматически создаст все необходимые таблицы.

### 4. Вход в систему

После установки перейдите на страницу входа:

- **Логин:** `admin`
- **Пароль:** `admin`

⚠️ **Важно:** Смените пароль после первого входа!

## Цветовая схема (Texmobile style)

| Элемент | Цвет |
|---------|------|
| Primary | `#e67e22` (оранжевый) |
| Primary Dark | `#d35400` |
| Background | Технический чертежный паттерн |
| Header | Белый |
| Navbar | Градиент оранжевый |
| Cards | Белый с тенью |
| Text | `#34495e` (темно-серый) |

## Статусы ремонта

- **Новая заявка** — только создана
- **Диагностика** — проводится диагностика
- **В работе** — ремонт выполняется
- **Ожидание запчастей** — ожидание комплектующих
- **Готов к выдаче** — ремонт завершен
- **Выдан клиенту** — устройство выдано
- **Отменен** — заявка отменена

## Роли пользователей

| Роль | Доступ |
|------|--------|
| `admin` | Полный доступ, управление устройствами |
| `manager` | Создание и редактирование заявок |
| `technician` | Просмотр и обновление статусов |

## Безопасность

- ✅ Пароли хранятся в хешированном виде (password_hash)
- ✅ Защита от SQL-инъекций через PDO prepared statements
- ✅ Защита от XSS через htmlspecialchars
- ✅ Проверка сессии на всех защищенных страницах
